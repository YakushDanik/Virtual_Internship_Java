1) clean (опионально) - чистит папку build для чистой среды сборки
2) compile - компилирует джава код
3) processResourse - обработка всех ресурсов в папке resource
4) classes - обхединяет результаты компиляции и обработки ресурсов
5) test - компилирует и запускает тесты
6) сборка в jar-файл
7) check - проверка качества данных
8) завершение build, итоговые артефакты попадают в папку build
